试用VLESS手机倒车摄像头

ZExceed / 2014-02-13 19:51:30 / 杂谈

一直对车载应用都挺感兴趣，正好看到一篇创业软文，春节闲着就给他们发了封建议信，节后就收到了试用产品，到手第一件事情就是拆、然后了解实现原理

原理

  • 摄像头通电，发射一个Wifi热点，手机Wifi连接上去，形成局域网，APP获取摄像头画面。

安装

查看手册发现保险丝21号未用，就接了条线出来测试下

倒车步骤

  • 上车->激活手机->开启APP->显示画面（除首次外WIFI是自动连接的）->挂倒档倒车->倒车完成->关闭APP->关闭手机。

这里就存在一个较大硬伤问题:

  1. 倒车步骤太复杂，启动延迟久，经过我试用点击APP到看到画面差不多需要3-5秒，所以在倒车功能上，是无法与如今车载相比的，尽管清晰度上占优势，但操作便捷是最重要的。
  2. 当手机连上摄像头（自动），手机就无法上网，无法收到微信之类APP的消息，也无法使用需要联网的APP，此时只能关闭WIFI。
  3. 安装需要打孔（附送钻头），但不是人人都有电钻这样大型工具的，只能由维修厂安装了。

因此目前我并没有安装在车上，或许用于监控动植物、鱼缸、门铃监控等等。

使用淘宝IP地址库获取IP归属地信息

ZExceed / 2013-03-11 13:20:33 / 知识

之前一直用QQWrt（纯真IP库），老忘记更新升级，经常有用户反馈IP错归属错误。

今天找了几个在线IP库，发现淘宝这个IP不错对比了几条信息都满准确，相信稳定性应该没问题，而且请求限制10QPS足够了。
地址：http://ip.taobao.com/

请求接口（GET）

  • http://ip.taobao.com/service/getIpInfo.php?ip=[ip地址字串]

响应信息

  • （json格式的）国家 、省（自治区或直辖市）、市（县）、运营商

返回数据格式

  • {“code”:0,”data”:{“ip”:”210.75.225.254”,”country”:”\u4e2d\u56fd”,”area”:”\u534e\u5317”, “region”:”\u5317\u4eac\u5e02”,”city”:”\u5317\u4eac\u5e02”,”county”:””,”isp”:”\u7535\u4fe1”, “country_id”:”86”,”area_id”:”100000”,”region_id”:”110000”,”city_id”:”110000”, “county_id”:”-1”,”isp_id”:”100017”}}
  • 其中code的值的含义为，0：成功，1：失败。

用了几天还蛮稳定。

华为HG553刷OPENWRT设置10M全双工

ZExceed / 2013-01-18 22:45:09 / 知识

我所在是老小区，当地最早通LAN，但是需要设置网卡10M全双工拨号才能达到真正10M网速（实际下载1.2M/s），自适应模式的话实际下载不超过700K/s而且不稳定，市面上很难找到可以设置10M全双工的路由器，正好有台老路由HG553，查了些资料可以刷OPENWRT那么也就能设置网卡工作模式了。

相关资料：http://wiki.openwrt.org/toh/huawei/hg553

  1. 首先下载编译好的固件（HW553通用，有经验也可以自己编译）
  2. 按住背后的researt键不放开，插上电源等15秒，然后浏览器打开192.168.1.1根据提示刷入固件
  3. 刷机完成后，使用telnet登录 telnet 192.168.1.1 修改root密码 passwd root 此时telnet服务自动禁用，ssh自动开启
  4. 由于HG553是没有WAN拨号功能的（这是ADSL集成猫路由器），所以需要虚拟一个WAN绑定到其中一个LAN口上，上面资料中配置文件里是绑定到第四个LAN口，不过不知道为什么后面无法设置网卡工作模式，所以我给调整到第一个LAN口上。

然后XSHELL等工具SSH连接上去重启网络服务 /etc/init.d/network restart

请确保宽带进线插LAN1口，自己电脑网线插2-4口，此时应该能连上网了，不行就重启下路由器。

然后再安装路由器WEB管理界面LUCI：

此时可以打开浏览器用root账户密码登录进去设置简体中文、开启无线等等。

下面如果你不需要设置10M全双工请跳过

  1. 下载 mii-tool工具并用WINSCP上传至路由器（其它路由器下载对应的包即可）
  2. 下载 http://downloads.openwrt.org/snapshots/trunk/brcm63xx/packages/ 搜索 mii-tool 下载最新版
  3. 安装 opkg install mii-tool_最新版本号_brcm63xx.ipk
  4. 执行 mii-tool -F 10BaseT-FD eth1这样第一个LAN口就强制10M全双工了。

可以到LUCI界面的网络-交换机中确定是否生效

终于爽爆了，不能接入光纤的老小区伤不起呀。

CS游戏的天梯系统设计思考

ZExceed / 2012-10-15 20:30:30 / 分享

传统竞技游戏，玩家玩的越久、打怪越多，那么积分自然越高，但决不代表玩家的水平也如积分那么高，因而久玩乏味，所以11平台开发了DOTA天梯系统这是他成功的关键因素。

天梯系统的几个关键点：
强弱关系

  • 对手比你强（天梯比你高）你胜天梯加多分，输天梯减少分。
  • 你比对手强（天梯比他低）你胜天梯加少分，输天梯减多分。
  • 加减分多少与你和对手的差距有比较大关系，其思想是高手赢了新手，理所当然，输给新手，不可原谅，反之亦然。

战绩表现

  • 从CS上讲，你是爆头多？C4多？1V3？1V4？还是K/D比多寡？战绩也决定该场加减分情况，即便你在这场比赛里是输的一方，但有可能你是该场的MVP，照样得加分。

    根据HLTV.ORG等网站历史DEMO战绩分析得出的三个平均比率：

    1. 平均每局杀人数=0.679
    2. 平均生存比率=0.317
    3. 平均多杀比率=1.277

Rating计算公式

  • Rating =（杀人Rating + 0.7×生存Rating + 多杀Rating）/ 2.7
  • 杀人Rating = 杀人数/局数/平均每局杀人数
  • 生存Rating =（局数—死亡数）/局数/平均生存比率
  • 多杀Rating =（1kill局数+4x2kill局数+9x3kill局数 + 16x4kill局数 + 25*5kill局数）/总局数/平均多杀比率

经过5E平台N个场次比赛日志分析，得出普通选手平均正常表现的Rating值是0.8，因此假设当选手这场比赛Rating>0.8就是好的表现，哪怕是输的，<0.8表现太差了，哪怕是赢得，都得减分。

此时，会有个问题，如果这个选手今天状态不佳输的太惨了，扣了许多分那之前表现不都白忙活了，所以我们要定义一个积分区间，每个天梯积分等级都要有自己的积分增减区间。

例如：

经过大量历史参比数据可以得出这样的公式：

  • 如果 Rating < 0.8 积分 = 积分区间负数 – (积分区间负数/0.8* Rating)
  • 如果 Rating = 0.8 积分 = 0 （没表现不加不减）
  • 如果 Rating > 0.8 积分 = 积分区间正数/(2-0.8)*( Rating-0.8)
